Chapter 9 — Project Scheduling
BMGT 300 · Networks, Duration Estimation, Critical Path
Definition
Project schedule
An output of a schedule model that presents linked activities with planned dates, durations, milestones, and resources (PMBoK).
Definition
Project planning
Identification of project objectives and the ordered activities needed to complete the project, including resource types and quantities.
Key network types
AON vs AOA
AON (Activity-on-Node): Nodes = activities, arrows = dependencies.
AOA (Activity-on-Arrow): Arrows = activities, nodes = events (start/finish).
Critical concepts
Float & Critical Path
Float (slack): time an activity can be delayed without delaying the project.
Critical path: longest path through the network; activities on it have zero total float.
Key scheduling terminology
Term Definition
Path Sequence of activities defined by the network logic
Event A point when an activity is started or completed
Node Junction point joined by dependency lines
Predecessors Activities that must be completed before a later activity can start
Successors Activities that cannot start until previous ones are completed
Merge activity An activity with two or more immediate predecessors
Burst activity An activity with two or more immediate successors
Float (slack) Amount of time an activity can be delayed without delaying the project
Critical Path Method (CPM) Network analysis technique to determine schedule flexibility and minimum project duration
Activity-on-Node (AON) fundamentals
Node label
Standard AON node
Each node shows: Early Start (ES), Early Finish (EF), Late Start (LS), Late Finish (LF), Activity ID, Duration. The forward pass fills ES/EF, backward pass fills LS/LF.
Logic
Serial vs Concurrent
Serial activities flow one after another. Concurrent activities can happen at the same time, creating parallel paths through the network.
Structure
Merge activity
An activity that cannot start until multiple predecessors finish. Example: Activity D depends on A, B, and C.
Structure
Burst activity
An activity that, once completed, allows multiple successors to begin. Example: Activity A releases B, C, and D simultaneously.
Duration estimation methods
Method 1
Past experience
Use history from similar projects as a guide.
Method 2
Expert opinion
Consult a past project manager or expert for their judgment.
Method 3
Mathematical derivation (Beta distribution)
Use three time estimates: a = optimistic, m = most likely, b = pessimistic.
Beta distribution formulas
Expected time (TE)
TE = (a + 4m + b) / 6
Activity variance (σ²)
σ² = ((b − a) / 6)²
Project variance & standard deviation
Project σ² = Σ variances on critical path
Project σ = √(project variance)
Example (Project Delta, critical path: A–C–D–F–H)
Activity a m b TE Variance
A 3 5 11 5.33 1.78
C 4 6 8 6 1.00
D 8 10 12 10 4.00
F 3 4 5 4 0.69
H 2 3 4 3 0.25
Project variance = 1.78 + 1 + 4 + 0.69 + 0.25 = 7.72 · Std dev = √7.72 = 2.78
Constructing the critical path
Step 1
Forward pass
Additive move from start to finish. Determines Earliest Start (ES) and Earliest Finish (EF). The finish EF gives the overall project duration.
Step 2
Backward pass
Subtractive move from finish to start. Determines Latest Start (LS) and Latest Finish (LF). Begins with the project end date as the LF of the last activity.
Step 3
Calculate float
Float = LS − ES (or LF − EF). Activities with zero total float are on the critical path.
Step 4
Identify critical path
The path with the longest total duration and zero float. Determines the minimum project duration.
Probability of finishing on time
Z‑score formula
Z = (Due date − Expected completion) / Project standard deviation
Look up Z in the standard normal table to find the probability of finishing by the due date.
Example: If expected completion = 30 weeks, due date = 32 weeks, σ = 2.78, then Z = (32−30)/2.78 = 0.72 → probability ≈ 76.42%.
